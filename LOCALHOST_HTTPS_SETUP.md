# 🔒 Localhost HTTPS Setup Guide
## For GolfGuiders Web App Development

---

## Why HTTPS on Localhost?

The centralized auth portal (`auth-dev.golfguiders.com`) sets cookies with the `Secure` flag. These cookies can **only** be accessed via HTTPS connections. Running on plain HTTP (`http://local.app.golfguiders.com:3000`) will result in authentication failures because the browser cannot read the secure cookies.

**Solution:** Run localhost with HTTPS using self-signed certificates.

- ✅ Node.js and Yarn/NPM installed
- ✅ Project cloned from repository

## Step 1: Update Hosts File

Add local subdomain to your system's hosts file:

### Windows
1. Open Notepad **as Administrator**
2. Open file: `C:\Windows\System32\drivers\etc\hosts`
3. Add this line:
   ```
   127.0.0.1   local.app.golfguiders.com
   ```
4. Save and close

### macOS / Linux
1. Open terminal
2. Edit hosts file:
   ```bash
   sudo nano /etc/hosts
   ```
3. Add this line:
   ```
   127.0.0.1   local.app.golfguiders.com
   ```
4. Save (`Ctrl+O`, `Enter`) and exit (`Ctrl+X`)

## Step 2: Install mkcert

### Windows

**Option A: Using Chocolatey**
```powershell
choco install mkcert
```

**Option B: Using Scoop**
```powershell
scoop bucket add extras
scoop install mkcert
```

**Option C: Manual Download**
1. Download: [mkcert-v1.4.4-windows-amd64.exe](https://github.com/FiloSottile/mkcert/releases/download/v1.4.4/mkcert-v1.4.4-windows-amd64.exe)
2. Open **PowerShell as Administrator**
3. Run:
   ```powershell
   Move-Item "$env:USERPROFILE\Downloads\mkcert-v1.4.4-windows-amd64.exe" "C:\Windows\System32\mkcert.exe"
   ```

### macOS

**Using Homebrew:**
```bash
brew install mkcert
brew install nss # For Firefox support (optional)
```

### Linux (Debian/Ubuntu)

```bash
sudo apt install libnss3-tools
curl -Lo mkcert https://github.com/FiloSottile/mkcert/releases/download/v1.4.4/mkcert-v1.4.4-linux-amd64
chmod +x mkcert
sudo mv mkcert /usr/local/bin/
```

## Step 3: Install Local Certificate Authority

Run this command to install mkcert's root CA in your system trust store:

```bash
mkcert -install
```

**You'll see a popup asking for permission - click Yes/Allow.**

This makes all certificates generated by mkcert trusted by your browser.

---

## Step 4: Generate SSL Certificates

Navigate to the project root and generate certificates:

### Windows (PowerShell)
```powershell
cd C:\xampp\htdocs\golfguiders-web-app
mkcert local.app.golfguiders.com localhost 127.0.0.1 ::1
New-Item -ItemType Directory -Force -Path .certs
Move-Item "local.app.golfguiders.com+3.pem" ".certs\cert.pem" -Force
Move-Item "local.app.golfguiders.com+3-key.pem" ".certs\key.pem" -Force
```

### macOS / Linux (Terminal)
```bash
cd /path/to/golfguiders-web-app
mkcert local.app.golfguiders.com localhost 127.0.0.1 ::1
mkdir -p .certs
mv local.app.golfguiders.com+3.pem .certs/cert.pem
mv local.app.golfguiders.com+3-key.pem .certs/key.pem
```

**What this does:**
- Generates SSL certificate for `local.app.golfguiders.com`, `localhost`, `127.0.0.1`, and `::1`
- Creates `.certs` directory (already in `.gitignore`)
- Organizes certificates with standard names

## Step 5: Verify Environment Configuration

The repository should already have these files configured:

### ✅ `.env.local` (Localhost)
```env
NEXT_PUBLIC_ENV=local
NEXT_PUBLIC_APP_DOMAIN=https://local.app.golfguiders.com:3000
NEXT_PUBLIC_AUTH_URL=https://auth-dev.golfguiders.com/auth/login?app_client_id=gg-app&redirect_uri=https://local.app.golfguiders.com:3000
NEXT_PUBLIC_AUTH_LOGOUT_URL=https://auth-dev.golfguiders.com/auth/logout?app_client_id=gg-app&redirect_uri=https://local.app.golfguiders.com:3000
```

### ✅ `package.json` (HTTPS Script)
```json
{
  "scripts": {
    "dev:https": "next dev --experimental-https --experimental-https-key .certs/key.pem --experimental-https-cert .certs/cert.pem"
  }
}
```

**If these are missing or outdated, update them according to the examples above.**

## 🛠️ Troubleshooting

### Issue 1: "Your connection is not private" Warning

**Cause:** Certificate not trusted by browser

**Solution:**
1. Make sure you ran `mkcert -install`
2. Restart your browser completely
3. Clear browser cache
4. Try accessing the site again

**For Chrome/Edge:** Click "Advanced" → "Proceed to local.app.golfguiders.com"

### Issue 2: "Cannot find certificates" Error

**Cause:** Certificates not generated or in wrong location

**Solution:**
```bash
# Verify certificates exist
ls -la .certs/
# Should show: cert.pem and key.pem

# If missing, regenerate:
mkcert local.app.golfguiders.com localhost 127.0.0.1 ::1
mkdir -p .certs
mv local.app.golfguiders.com+3.pem .certs/cert.pem
mv local.app.golfguiders.com+3-key.pem .certs/key.pem
```

### Issue 3: Still Getting Redirected After Login

**Possible Causes:**

1. **Auth portal redirect URI not whitelisted**
   - Contact backend team to add `https://local.app.golfguiders.com:3000` to allowed redirect URIs

2. **Cookies still not accessible**
   - Visit debug page: `https://local.app.golfguiders.com:3000/en/debug-cookies`
   - Check if `golfguiders.auth-token` is present
   - If missing, check browser console for errors

3. **Environment variables not loaded**
   - Restart dev server after changing `.env.local`
   - Verify with: `console.log(process.env.NEXT_PUBLIC_APP_DOMAIN)`

### Issue 4: "Port 3000 already in use"

**Solution:**
```bash
# Find process using port 3000
# Windows:
netstat -ano | findstr :3000

# macOS/Linux:
lsof -ti:3000

# Kill the process or use different port:
yarn dev:https -- -p 3001
```

**Don't forget to update `.env.local` redirect URI if using different port!**

### Issue 5: Certificate Expired (After 365 Days)

**Solution:**
```bash
# Delete old certificates
rm -rf .certs/

# Regenerate
mkcert local.app.golfguiders.com localhost 127.0.0.1 ::1
mkdir .certs
mv local.app.golfguiders.com+3.pem .certs/cert.pem
mv local.app.golfguiders.com+3-key.pem .certs/key.pem

# Restart dev server
yarn dev:https
```

## 🔐 Security Notes

### For Development Only

⚠️ **These certificates are ONLY for local development.** They are:
- Self-signed and not trusted outside your machine
- Valid for 825 days (mkcert default)
- Stored in `.certs/` which is in `.gitignore`

### Never Commit Certificates

The `.gitignore` already excludes:
```
*.pem
.certs/
```

**Never commit certificates to git!** Each developer generates their own.

### Production

Production uses real SSL certificates from:
- Let's Encrypt (via Vercel/hosting provider)
- AWS Certificate Manager
- Or similar trusted CA
